# CRM 系统安装部署文档

**版本**：v1.0.0

---

## 一、环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Node.js | 16.x | 18.x LTS 或 20.x LTS |
| npm | 7.x | 9.x+ |
| 操作系统 | Windows 10 / macOS 12 / Ubuntu 20.04 | — |
| 内存 | 2GB | 4GB+ |
| 磁盘 | 500MB 可用空间 | 1GB+ |

> **注意**：无需安装数据库，系统使用 SQLite 内嵌数据库，首次启动自动创建。

---

## 二、快速启动（开发环境）

### 第一步：安装后端依赖

```bash
cd crm-system/backend
npm install
```

### 第二步：配置后端环境变量

```bash
cp .env.example .env
```

编辑 `.env` 文件（默认值可直接使用）：

```env
PORT=3001
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_EXPIRES_IN=7d
DB_PATH=./crm.db
```

### 第三步：启动后端服务

```bash
npm start
```

启动成功后输出：
```
🚀 CRM Server running on port 3001
📦 Database initialized successfully
```

### 第四步：安装前端依赖

打开新终端：

```bash
cd crm-system/frontend
npm install
```

### 第五步：启动前端开发服务

```bash
npm run dev
```

启动成功后输出：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
```

### 第六步：访问系统

打开浏览器访问：**http://localhost:5173**

登录账号：
- 用户名：`admin`
- 密码：`Admin@123`

## 三、生产环境部署

### 3.1 构建前端静态文件

```bash
cd crm-system/frontend
npm run build
```

构建完成后，静态文件位于 `frontend/dist/` 目录。

### 3.2 使用 Express 托管前端（推荐）

在 `backend/src/index.js` 中已预留静态文件托管逻辑，可将 `dist` 目录复制到 backend 旁边：

```bash
# 在项目根目录执行
cp -r frontend/dist backend/public
```

后端已配置：当访问非 API 路径时，返回 `public/index.html`。

完整生产启动：

```bash
cd backend
NODE_ENV=production node src/index.js
```

访问 `http://服务器IP:3001` 即可。

### 3.3 使用 Nginx 反向代理（推荐生产）

**nginx.conf 示例**：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /var/www/crm/frontend/dist;
    index index.html;

    # 前端路由（SPA）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 60s;
    }
}
```

### 3.4 使用 PM2 管理后端进程

```bash
# 全局安装 PM2
npm install -g pm2

# 启动服务
cd backend
pm2 start src/index.js --name crm-server

# 设置开机自启
pm2 startup
pm2 save

# 常用命令
pm2 status           # 查看状态
pm2 logs crm-server  # 查看日志
pm2 restart crm-server  # 重启
pm2 stop crm-server     # 停止
```

## 四、Docker 部署

### 4.1 后端 Dockerfile

在 `backend/` 目录创建 `Dockerfile`：

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3001
CMD ["node", "src/index.js"]
```

### 4.2 docker-compose.yml

在项目根目录创建：

```yaml
version: '3.8'
services:
  backend:
    build: ./backend
    ports:
      - "3001:3001"
    volumes:
      - ./data:/app/data
    environment:
      - NODE_ENV=production
      - PORT=3001
      - JWT_SECRET=change-this-in-production
      - DB_PATH=/app/data/crm.db
    restart: unless-stopped

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./frontend/dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend
    restart: unless-stopped
```

### 4.3 启动

```bash
# 构建前端
cd frontend && npm run build && cd ..

# 启动容器
docker-compose up -d

# 查看日志
docker-compose logs -f
```

## 五、数据库说明

### 数据库位置

SQLite 数据库文件位于 `backend/crm.db`（由 `DB_PATH` 环境变量控制）。

### 数据备份

```bash
# 简单备份
cp backend/crm.db backup/crm_$(date +%Y%m%d).db

# 定时备份（Linux crontab）
0 2 * * * cp /path/to/crm.db /backup/crm_$(date +\%Y\%m\%d).db
```

### 初始化说明

首次启动时，系统自动执行以下操作：
1. 创建所有数据表
2. 插入默认角色（超级管理员/销售经理/销售顾问/项目经理）
3. 创建 admin 账号（密码：Admin@123）
4. 插入演示数据（客户、线索、商机、合同、回款、项目）

### 重置数据库

```bash
# 停止服务后删除数据库文件，重启即可重新初始化
rm backend/crm.db
npm start  # 重新启动后自动创建
```

## 六、环境变量说明

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | `3001` | 后端监听端口 |
| `JWT_SECRET` | `crm-secret-key` | JWT 签名密钥，**生产环境必须修改** |
| `JWT_EXPIRES_IN` | `7d` | Token 有效期（支持 `1h`, `7d`, `30d` 等） |
| `DB_PATH` | `./crm.db` | SQLite 数据库文件路径 |
| `NODE_ENV` | `development` | 运行环境（`production` 会关闭详细日志） |

## 七、API 文档

### 认证方式

所有 API（除登录接口外）需在请求头中携带 JWT Token：

```
Authorization: Bearer <token>
```

### 接口列表

#### 认证接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 用户登录 |
| GET | `/api/auth/profile` | 获取当前用户信息 |
| PUT | `/api/auth/profile` | 更新个人信息 |
| PUT | `/api/auth/password` | 修改密码 |

#### 业务接口（均支持 GET/POST/PUT/DELETE）
| 路径前缀 | 模块 |
|---------|------|
| `/api/leads` | 线索管理 |
| `/api/customers` | 客户管理 |
| `/api/opportunities` | 商机管理 |
| `/api/projects` | 项目管理 |
| `/api/contracts` | 合同管理 |
| `/api/payments` | 回款管理 |
| `/api/reports/*` | 报表数据 |
| `/api/system/users` | 用户管理 |
| `/api/system/roles` | 角色管理 |
| `/api/system/logs` | 操作日志 |

#### 健康检查
```
GET /api/health
```

## 八、常见问题

### Q: 启动时报 `better-sqlite3` 编译错误

```bash
npm install --build-from-source better-sqlite3
# 或者确保安装了 node-gyp 依赖
npm install -g node-gyp
```

Windows 用户还需安装：
```bash
npm install -g windows-build-tools
```

### Q: 前端无法连接后端 API

检查 `frontend/vite.config.js` 中代理配置：

```js
proxy: {
  '/api': { target: 'http://localhost:3001', ... }
}
```

确保后端在 3001 端口正常运行：`curl http://localhost:3001/api/health`

### Q: 登录时提示"用户名或密码错误"

确认使用的是默认账号：用户名 `admin`，密码 `Admin@123`（注意大小写和特殊字符）。

### Q: Token 过期后页面空白

清除浏览器 localStorage 后重新登录：

```javascript
// 在浏览器控制台执行
localStorage.clear()
location.reload()
```

### Q: 数据库锁定错误（SQLITE_BUSY）

SQLite 同时只支持一个写入连接。如并发需求高，建议切换到 MySQL 或 PostgreSQL。

## 九、升级指南

```bash
# 1. 备份数据库
cp backend/crm.db backup/crm_before_upgrade.db

# 2. 拉取新代码
git pull origin main

# 3. 更新后端依赖
cd backend && npm install

# 4. 更新前端依赖并重新构建
cd ../frontend && npm install && npm run build

# 5. 重启服务
pm2 restart crm-server
```

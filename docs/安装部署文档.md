# CRM 系统 v2.0 安装部署文档

## 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Node.js | 16.x | 18.x LTS 或 20.x LTS |
| npm | 7.x | 9.x+ |
| 操作系统 | Windows 10 / macOS 12 / Ubuntu 20.04 | — |

> **注意**：无需安装数据库，系统使用 SQLite 内嵌数据库，首次启动自动创建。

## 快速启动（开发环境）

### 第一步：启动后端

```bash
cd crm-system/backend
npm install
cp .env.example .env
npm start
```

启动成功后输出：
```
🚀 CRM Server running at http://localhost:3001
✅ 数据库初始化完成
```

### 第二步：启动前端

打开新终端：

```bash
cd crm-system/frontend
npm install
npm run dev
```

### 第三步：访问系统

打开浏览器访问：**http://localhost:5173**

## 常见问题

### Q: 安装时 better-sqlite3 编译失败

```bash
# 方法1: 安装 build-essential
sudo apt-get install build-essential python3-dev  # Linux
xcode-select --install  # macOS

npm install --build-from-source better-sqlite3

# 方法2: 使用 node-gyp
npm install -g node-gyp
npm install
```

### Q: 前端无法连接后端

检查 `frontend/vite.config.js` 中代理配置：

```js
proxy: {
  '/api': { target: 'http://localhost:3001', changeOrigin: true }
}
```

确保后端在 3001 端口正常运行：
```bash
curl http://localhost:3001/api/health
```

### Q: 重置数据库

```bash
# 停止服务后删除数据库文件，重启即可重新初始化
rm backend/crm.db
npm start  # 重新启动后自动创建并填充示例数据
```

## 生产环境部署

### 使用 PM2 管理后端进程

```bash
npm install -g pm2
cd backend
pm2 start src/index.js --name crm-server
pm2 startup && pm2 save
```

### 构建前端静态文件

```bash
cd frontend
npm run build
# 静态文件位于 frontend/dist/
```

### Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /var/www/crm/frontend/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
